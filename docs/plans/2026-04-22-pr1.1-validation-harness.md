## 1. Problem statement

PR1 shipped the v0.1.0 extractor baseline but **did not gate quality**. PR1.2 (wheel publication) is blocked on per-doc-type precision/recall thresholds from parent plan §H8. This PR delivers the **measurement infrastructure** so those thresholds can be computed once real labeled PDFs are in place.

## 2. Scope split

PR1.1 has two halves:
- **(a) the harness** — test code that loads labels.yaml + per-type thresholds + scorecard emission
- **(b) the labels** — 30 real PDFs hand-labeled per H8 stratification

This PR ships **(a) only.** Half (b) is a human-labeling task that requires real Army PDFs and judgment calls; tracked as a follow-up task.

## 3. Approach

- `validation_set/README.md` — the contract: where labels live, how to add them, per-type thresholds.
- `validation_set/labels.example.yaml` — format reference, NOT real labels (committed).
- `validation_set/labels.yaml` — gitignored until full set is ready.
- `validation_set/pdfs/` — gitignored (real PDFs are large + sensitive).
- `tests/test_validation_set.py` — opt-in pytest under `@pytest.mark.validation`. Loads labels.yaml + runs extractor against each PDF + computes per-type precision/recall + def-text Jaccard. Fails with a per-type scorecard if any gate is breached. Emits `validation_set/scorecard.json` for PR1.2 to reference.
- `pytest.ini` config: default test runs exclude the `validation` marker; CI doesn't fail without labels.

## 4. Hard 30%

- **H1 — label sample bias.** Labeled set is sampled (~5 defs/PDF), not exhaustive. False-positive count is therefore conservative-overcount; thresholds are calibrated for labeled coverage. Acceptable; documented in test docstring.
- **H2 — Jaccard token-overlap is a coarse measure.** Real definition correctness has nuance (truncation, paraphrasing). Thresholds are intentionally generous (≥75–85% Jaccard, not 100%). Refinement deferred.
