# v0.5 Cohort Eligibility Cleanup — Class-4 Short-Doc Exclusion

**Date:** 2026-05-18
**Branch:** `chore/2026-05-18-v0.5-cohort-eligibility`
**Roadmap parent:** `docs/plans/2026-05-18-v0.5-roadmap-definition-extraction.md`

## Context

D-2's findings memo identified **Class 4** of the v0.5 residual cohort: **7 of 45 docs** were short docs (<10 pages) where the word "glossary" appears in passing (footnote, cross-reference) but the doc has no real glossary section. These docs were captured in the original "zero-entry-with-glossary" cohort definition by the SQL predicate:

```sql
WHERE q.status='SUCCEEDED' AND q.extractor_version='army-v2.1.0' AND q.entry_count = 0
  AND d.extracted_text_status='HAS_TEXT'
  AND d.extracted_text ILIKE '%glossary%'
  AND length(d.extracted_text) > 100000;
```

The `length > 100000` floor was intended to exclude short docs, but Class-4 docs include some with substantial text but few pages OR short docs that happen to have substantial OCR'd text content. The 7 Class-4 docs aren't real "extraction failures" — they're docs with no glossary to extract.

## Fix

Update the cohort-eligibility predicate to add a page-count floor of **30 pages** (heuristic: Army Pubs with real glossaries are reliably >30 pages). Document the predicate at `docs/ops/extraction-cohort-definitions.md` (new file) for future re-extraction waves to reference.

## Scope

**Docs-only change.** Updates a reference document; no code changes; no production data mutation. The next operator-UPDATE re-extraction wave applies the new predicate.

The 7 Class-4 docs from the v0.5 residual cohort remain in production as `SUCCEEDED, entry_count=0` — that's correct for them. They simply won't appear in future "failing-cohort" predicates that use the new definition.

## Verification

- `docs/ops/extraction-cohort-definitions.md` exists and is referenced by the v0.5.0 release-gate plan.
- No code, test, or schema changes.

## Out of scope

- Re-running the cohort SQL against production (operator action, not docs).
- Modifying the 7 Class-4 docs' DB rows.
- Changing the queue schema.
