**Date:** 2026-05-17 (revised iter-3)
**Branch:** `feat/2026-05-18-v0.5-unit0` (continuing from Unit 0)
**Roadmap parent:** `docs/plans/2026-05-18-v0.5-roadmap-definition-extraction.md`

## Context

Unit 0 split the v0.4 residual cohort (45 docs) by failure mode. D-1 addresses **27 docs** where v0.4's `find_glossary_page_range` returned a wrong range:

- **Class 2 (21 docs):** range overlaps actual glossary but is mis-bounded (typically truncates to last 2 pages).
- **Class 3 (6 docs):** range disjoint from actual glossary (typically a body-text reference).

The remaining 18 docs are out of scope: 11 are Class 1 (correct range, parser bug → D-2); 7 are Class 4 (short docs without real glossaries → cohort-eligibility cleanup).

## Root cause (empirically verified)

`find_glossary_page_range` (`extractors/glossary.py:175-211`) uses a backward-sweep-first-match-wins strategy capped at `MAX_GLOSSARY_LOOKBACK_PAGES = 75`. Empirical trace of ATP 3-21.10 shows the glossary header regex matches on EVERY page of the section (pages 580-604 = 25 contiguous matching pages, because Army doctrine prints the section name as a running-header footer). The backward sweep returns page 604 — the LAST page of the glossary — and the parser sees only 2 pages instead of 25. Same shape for the 20 other Class-2 docs.

For Class 3, the doc has an isolated body-text "see Glossary" or sidebar reference; the backward sweep finds it before reaching the real glossary section earlier in the doc.

## The decisive design choice

**One rule, stated once, reflected everywhere:**

> Scan the entire doc forward. Group consecutive matching pages into contiguous blocks. Pick the largest block. **Tie-break: prefer the EARLIER block** — handles the "real glossary + later back-cover symbols sidebar" case where the real glossary is earlier and the sidebar is later.

This single rule:
- Fixes Class 2 (largest block IS the real glossary)
- Fixes Class 3 (multi-page real glossary outranks isolated body-text matches)
- Removes the need for `MAX_GLOSSARY_LOOKBACK_PAGES` (full-doc scan obviates the cap)

## Approach

### Decisions taken (avoiding the iter-1/2 layer-cake)

| Decision | Choice | Rationale |
|---|---|---|
| Tie-break | **EARLIER block wins** | A back-cover sidebar appears later than the real glossary. Earlier-wins handles both single-largest-block and equal-sized-blocks correctly. |
| Gap tolerance | **Strict contiguous (no gaps)** by default; verified at pre-implementation by empirical scan. If the scan finds ≥1 split block on a real validation-set doc, switch to 1-page-gap tolerance. |
| `MAX_GLOSSARY_LOOKBACK_PAGES` | **Remove entirely** | Full-doc scan removes the need; test asserting removal added. |
| Version bump | **Not in this PR.** `pyproject.toml` stays at 0.4.0 + dev-suffix; CHANGELOG entry added but as `[unreleased]`. Version bump + release happen in a separate gate after D-2 (and Unit 3a, if applicable) land. |
| Acceptance precision | **Exact ranges per-doc** for Class 2+3 (27 docs). Tolerance allowed ONLY when documented per-doc in the acceptance YAML with rationale. Default: no tolerance. |

### Code shape (canonical — supersedes any prior sketch)

```python
# Replaces lines 175-211 of glossary.py
def find_glossary_page_range(doc, profile):
    total = len(doc)
    header_res = [re.compile(p, re.IGNORECASE | re.MULTILINE)
                  for p in profile.glossary_header_patterns]
    end_res = [re.compile(p, re.IGNORECASE | re.MULTILINE)
               for p in _GLOSSARY_END_PATTERNS]

    # Step 1: forward-scan, collect matching page indices.
    matching: list[int] = []
    for i in range(total):
        try:
            text = doc[i].get_text("text")
        except Exception:
            continue
        if any(r.search(text) for r in header_res):
            matching.append(i)
    if not matching:
        return None

    # Step 2: group into strict-contiguous blocks (no gap tolerance — see Hard 30% §G).
    blocks: list[list[int]] = []
    cur = [matching[0]]
    for p in matching[1:]:
        if p == cur[-1] + 1:
            cur.append(p)
        else:
            blocks.append(cur)
            cur = [p]
    blocks.append(cur)

    # Step 3: pick largest block; tie-break by EARLIER position.
    blocks.sort(key=lambda b: (-len(b), b[0]))
    found_start = blocks[0][0]

    # Step 4: end-scan forward from found_start (existing logic; whole-line
    # anchored end patterns make standalone terminator-as-glossary-term safe).
    end = total - 1
    for i in range(found_start + 1, total):
        text = doc[i].get_text("text")
        if any(r.search(text) for r in end_res):
            end = i - 1
            break
        if _is_back_cover_marker(text, i, total):
            end = i - 1
            break
    return (found_start, end)
```

## Hard 30%

### H1. Profile-pattern audit may surface a broad pattern
The audit (A1) classifies each pattern. If any pattern is not whole-line anchored (e.g., a future profile adds `\bGlossary\b`), full-doc scan would match body text. **Mitigation:** the audit fails-loud and blocks implementation. Today only `ArmyProfile` exists; all its patterns are `^...$` anchored.

### H2. End-scan truncation on standalone terminator-as-glossary-term
A glossary entry could be `"Appendix B — ..."` rendered as a standalone line. Under v0.4 (narrow start range), end-scan rarely crossed enough pages to encounter this. Under v0.5 (full-glossary range), the crossing distance is much larger. **Mitigation:** test in `test_standalone_terminator_as_glossary_term_does_not_break_end_scan` using ATP 3-21.10 fixture. If the test fails empirically, add a defensive check (next-line content validation) before triggering end-of-glossary.

### H3. Section II boundary shift
With v0.5's earlier `found_start`, the [start, end] range may now include Section I content the parser previously didn't see. `narrow_to_section_ii` should clip to Section II correctly, but the narrowed range could differ. **Mitigation:** `test_section_ii_narrowed_range_matches_section_ii_header` asserts exact Section II boundary; `test_section_ii_extracted_terms_stable` asserts no term-identity drift.

### H4. Class-1 baseline drift
The Class-1 reference is generated from commit `c4d1611` (the merged v0.4.0 release). If the worktree has uncommitted changes that affect `find_glossary_page_range`, the baseline could be wrong. **Mitigation:** A2's capture script does `git fetch && git rev-parse c4d1611` and exits if HEAD doesn't match; runs in a fresh subprocess to avoid in-memory pollution.

### H5. Gap-tolerance regression risk
Strict contiguous is the default. If A6's scan finds ≥1 affected doc, switch to 1-page tolerance. The switch changes behavior on the 31-doc validation set — golden-output regression test will catch any unintended shift.

### H6. Profile coverage for future profiles
The forward-scan strategy is profile-generic but the safety analysis is profile-specific (per A1). **Mitigation:** add a comment in `find_glossary_page_range` requiring re-audit when a new profile is added. (Future-work; not Unit D-1 scope.)

### H7. Acceptance misses must be categorized
3 of 27 may not produce the exact expected range. Each must be categorized with a disposition: `deferred_to_d2` (parser is the real issue), `scope_decision` (intentionally out of scope), or `known_failure_with_followup_unit`. **No anonymous failures.** (Codex iter-2 #3.)

### H8. Realistic-PDF fixtures vs real PDFs
Text-dump fixtures cover unit-test cases. Real-PDF tests cover the PDF extraction layer (PyMuPDF page numbering, OCR artifacts, font metadata). **Mitigation:** Class-1 + Class-2+3 + 31-doc validation tests ALL run against real PDFs in `/tmp/v05-unit0/pdfs/` and `validation_set/pdfs/` respectively. Text fixtures are SUPPLEMENTARY, not replacement. (Codex iter-2 #10.)

## Why the iter-1/2 attempts didn't pass

Iter-1: tie-break was self-contradictory (text said earlier, code said later); acceptance was entry-count-based not range-correctness-based; Class-1 preservation was assumed not enforced; profile audit was a grep proxy; gap tolerance was "spot-check at execution".

Iter-2: tie-break still inconsistent across sections (layer-cake); acceptance `overlap` tolerance hid range correctness; 24/27 threshold left 3 unexplained failures; Section II test was loose containment; golden allowed deviations inside the baseline file; version bump made this look release-ready without D-2.

This iter-3 rewrite: one rule stated once, range-correctness primary, every miss categorized, all artifacts pinned to a specific v0.4 commit, deviations in a separate reviewer-approved file, version bump deferred to release gate.
