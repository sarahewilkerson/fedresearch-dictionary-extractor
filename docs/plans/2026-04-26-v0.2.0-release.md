## 1. Problem statement

Units 1-4 of v0.2.0 are merged. Unit 5 publishes the wheel and refreshes the corpus state to reflect Unit 3's Section II scoping behavior (existing docs change output) plus Unit 4's 4 new Section-I-heavy docs. Two existing test infrastructures must be updated:

1. **`tests/test_labels_classifier.py::test_no_unexpected_classifier_flips`** — asserts `added == ∅` against the immutable prefix snapshot. Unit 4 added 4 new docs to candidate-output; their entries become an `added` set that must be tracked via a new `NEW_DOCS_SINCE_PREFIX` allowlist.
2. **`tests/test_batch1_reconciled.py::test_v0_1_0_corpus_pin_emits_known_forbidden_pair`** — asserts forbidden pairs ARE in candidate-output. Per its module docstring lifecycle, when extractor regen drops the bad pair, this PR must (a) invert assertion to `not in actual`, (b) replace with extractor-level test, OR (c) delete. Decision: **invert** — reuses existing infrastructure as the post-fix regression detector.

Plus the routine release work: bump version, regenerate candidate-output, refresh classifier snapshot, build wheel, CHANGELOG.

### 3.2 Regenerate candidate-output

Script-driven regeneration over all 30 local PDFs (27 in original + 3 added in Unit 4 + AR 380-381 added in Unit 3). Non-deterministic field stripped: `extraction_timestamp`.

```bash
.venv/bin/python <<'PY'
import json
from pathlib import Path
from fedresearch_dictionary_extractor.core.analyzer import analyze_pdf

ROOT = Path('/Users/mw/code/fedresearch-dictionary-extractor')
PDFS = sorted((ROOT / 'validation_set/pdfs').iterdir())
CO = ROOT / 'validation_set/candidate-output'
CO.mkdir(exist_ok=True)
for pdf in PDFS:
    if not pdf.suffix == '.pdf':
        continue
    try:
        out = analyze_pdf(str(pdf), profile_name='army')
    except Exception as e:
        print(f'SKIP {pdf.name}: {e}')
        continue
    if not out.get('entries'):
        # mirror existing convention: 0-entry docs get nothing in candidate-output
        # except the NO_DEFINITIONS sentinel (already present)
        continue
    out_path = CO / (pdf.stem + '.json')
    out.pop('extraction_timestamp', None)  # determinism
    out_path.write_text(json.dumps(out, indent=2, ensure_ascii=False, sort_keys=True), encoding='utf-8')
PY
ls validation_set/candidate-output/*.json | wc -l
```

**Verify:** count of resulting JSONs matches expected (27 + new docs that have entries; some "section_ii_only" + 0-entry docs are still 0).

- **Wheel build determinism**: `python -m build` produces a wheel with metadata derived from `pyproject.toml`. Version field MUST be 0.2.0 before build.
- **dist/ commit hygiene**: keep v0.1.0 wheel for backward reference (don't delete); add v0.2.0 alongside.
